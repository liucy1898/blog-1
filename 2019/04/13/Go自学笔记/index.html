Go自学笔记

发表于 2019-04-13 | 分类于 Go

新开的坑, 记录自学过程, 按照自己的学习顺序会一点一点补上

1. 变量

1.1 指针

  1. 基本数据类型, 变量存储的是值类型
  2. 通过 & 获取变量的地址
  3. 指针类型中变量存储的是一个地址, 这个地址指向的空间才是真正的值

2. 函数

2.1 闭包

闭包就是一个函数和与其相关的引用环境组合的一个整体

  1. AddUpper 是一个函数, 返回的数据类型是 func(int) int
  2. 返回的是一个匿名函数, 但这个匿名函数引用到了函数外的 n, 因此这个匿名函数就和 n 行程一个整体, 构成闭包.
  3. 可以把闭包理解成一个类, 而函数时方法, n 是字段
  4. 当闭包被反复调用的时候, 由于 n 只会初始化一次, 因此每调用一次就会被累加

闭包的功能更类似与面向对象中的类, 属性和方法, 闭包在多次调用中可以复用逻辑和一部分字段.

2.2 defer

在函数中, 经常需要创建资源(数据库连接, 文件句柄, 锁等), 为了在函数执行完毕后及时释放资源, Go 提供了 defer 的延时执行机制.

15. 单元测试

Go 语言中自带一个轻量级测试框架 testing 和自带的 go test 命令来实现单元测试和性能测试, testing 框架和其他语言中的测试框架类似, 可以基于这个框架写针对相应函数的测试用例, 也可以基于该框架写相应的压力测试用例

16. goroutine(协程) 和 channel(管道)

假设有如下需求:

统计 1 到 1000000000 中所有的质数
  • 传统的思路是使用一个循环, 将所有需要统计的数字由循环依次执行判断逻辑
  • 在 Go 中可以使用并行的方式, 将统计质数的任务分配给多个 goroutine 去完成

16.1 goroutine 基本介绍

  1. 进程就是程序在操作系统中的一次执行过程, 是系统进行资源分配和调度的最小单位.
  2. 线程是进程的一个执行实例, 是程序执行的最小单元, 它是比进程更小的能够独立运行的基本单位.
  3. 一个进程可以创建和销毁任意个线程, 一个进程中的多个线程可以并发执行.
  4. 一个程序至少有一个进程, 一个进程至少有一个线程.

Go 主线程可以启动多个协程, 协程可以理解为一种轻量级的线程, 特点:

  1. 独立的栈空间
  2. 共享程序堆空间
  3. 调度由用户控制
  4. 协程是轻量级的线程

16.2 简单案例

在主线程中开启一个 goroutine, 该协程每隔1秒输出 "goroutine"
在主线程中每隔 1 秒输出 "main", 输出 10 次后退出程序
要求主线程和 goroutine 同时执行

当使用 go 关键字去调用 test 方法的时候, 就会开启一个协程

16.3 MPG 模式

  • 主线程是一个物理线程, 直接作用在 CPU 上, 相对比较消耗 CPU 资源.
  • 协程由主线程创建, 是轻量级的逻辑态线程, 对资源的消耗相对更小.
  • Go 的协程机制是一个重要的特点, 可以轻松开启上万个协程, 其他编程语言的并发机制一般是基于线程的, 开启过多的线程资源消耗较大.

MPG 是 goroutine 的调度模型:

  • M: 操作系统的主线程
  • P: 协程执行需要的上下文
  • G: 协程

16.4 channel(管道)

假设现在有如下需求:

计算 1-20 的各个数字的阶乘, 并且把各个数的阶乘放入 map 中, 最后显示出来, 使用 goroutine 实现.

第一版代码比较简单, 但运行之后会发现两个严重的问题:

  1. 主线程大概率会在 200 个 goroutine 执行完毕之前结束
  2. goroutine 在执行的时候会报错: fatal error: concurrent map writes

因此引出了 goroutine 之间的通信问题, 有两种主要的解决方案:

  1. 全局变量加锁同步
  2. channel

16.5 使用全局变量加锁同步改进程序

由于没有对全局变量 map 加锁, 因此会出现资源争夺问题, 代码会出现错误, 因此加入互斥锁.
