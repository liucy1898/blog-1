Go自学笔记

发表于 2019-04-13 | 分类于 Go

新开的坑, 记录自学过程, 按照自己的学习顺序会一点一点补上

1. golang 工具链

1.1 go 工具/命令

go 自带一系列强大的命令, 其通用格式是

go command [arguments]

具体命令有:

命令 功能 备注
build compile packages and dependencies 最常用的命令之一, 编译 go 文件
isntall compile and install packages and dependencies 与 build 最大的区别是编译后会将输出文件打包成库放在 pkg 下
get download and install packages and dependencies 获取 go 的第三方包, 通常默认从 git repo 上 pull 最新的版本
fmt gofmt(reformat) package sources 统一代码风格和排版
test test packages 运行当前包目录下的 tests, golang 中 test 文件的命名为: main_test.go, 一般以 xxx_test.go 命名

1.2 Test 写法

假设有 main.go

以及它的测试类 main_test.go

  • 每个 test 文件必须 import 一个 testing 包;
  • test 文件下的每一个 test case 必须以 Test 开头并且符合 TestXxx 形式, 否则 go test 会直接跳过不执行. 其中 Xxx 一般约定为被测试的函数名;
  • t.Errorf 为打印错误信息, 并且当前 test case 会被跳过;
  • t.SkipNow() 为跳过当前 test, 并且直接按 PASS 处理下一个 test;
  • Go 不会严格保证多个 Test 是顺序执行, 可以用 t.Run 来执行 subtests 可以做到控制test 输出以及 test 的顺序
  • 使用 TestMain 作为初始化 test, 并且使用 m.Run() 来调用其他 test 可以完成一些需要初始化操作的 testing, 比如数据库连接, 文件等, 反之则除了 TestMain 以外的其他 tests 都不会被执行

2. 变量

2.1 指针

  1. 基本数据类型, 变量存储的是值类型
  2. 通过 & 获取变量的地址
  3. 指针类型中变量存储的是一个地址, 这个地址指向的空间才是真正的值

3. 函数

3.1 闭包

闭包就是一个函数和与其相关的引用环境组合的一个整体

  1. AddUpper 是一个函数, 返回的数据类型是 func(int) int
  2. 返回的是一个匿名函数, 但这个匿名函数引用到了函数外的 n, 因此这个匿名函数就和 n 行程一个整体, 构成闭包.
  3. 可以把闭包理解成一个类, 而函数时方法, n 是字段
  4. 当闭包被反复调用的时候, 由于 n 只会初始化一次, 因此每调用一次就会被累加

闭包的功能更类似与面向对象中的类, 属性和方法, 闭包在多次调用中可以复用逻辑和一部分字段.

3.2 defer

在函数中, 经常需要创建资源(数据库连接, 文件句柄, 锁等), 为了在函数执行完毕后及时释放资源, Go 提供了 defer 的延时执行机制.

15. 单元测试

Go 语言中自带一个轻量级测试框架 testing 和自带的 go test 命令来实现单元测试和性能测试, testing 框架和其他语言中的测试框架类似, 可以基于这个框架写针对相应函数的测试用例, 也可以基于该框架写相应的压力测试用例

16. goroutine(协程) 和 channel(管道)

Golang 对并发编程有着原生的支持, 在通用性语言中并不常见

代码的第 3 行中, 调用匿名函数之前加了一个关键字 go, 因此函数的调用会并发的执行, 一共会有 1000 个函数并发执行, 每个函数都会在死循环内打印一句话, 并且带上自己的 index

协程(Coroutine) 是一种轻量级线程, 所谓轻量级表现在:

  1. 非抢占式多任务处理, 由协程主动交出控制权. 传统的线程的控制权是由操作系统控制, 线程在任何时间点都有可能由于时间片结束而进入挂起状态.
  2. 编译器/解释器/虚拟机层面的多任务, 而不是操作系统层面的任务. 操作系统有自己的调度器, 而 go 也有自己的调度器来调度协程.
  3. 多个协程可能在一个或多个线程上运行.

16.1 goroutine 定义

  1. 任何函数只需加上 go 就能送给调度器运行
  2. 不需要在定义时区分是否是异步函数
  3. 调度器会在何时的点进行切换
  4. 使用 -race 进行数据冲突的检测

16.2 goroutine 可能的切换点

  1. I/O, select
  2. channel
  3. 等待锁
  4. 函数调用
  5. runtime.Gosched()

16.4 channel

channel 用于实现 goroutine 之间的通信
